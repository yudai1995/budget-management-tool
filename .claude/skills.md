# プロジェクト固有スキル定義

## branch

`main` ブランチにいる場合に、現在のタスクに基づいた新しいブランチを作成します。

- **Usage**: ユーザーの指示を受け取った直後に実行。
- **Constraint**: `feature/`, `fix/`, `refactor/` 等のプレフィックスを適切に使用すること。

## translate

日本語の指示を構造化して自己解析します。

- **Input**: ユーザーからの日本語指示
- **Output**: 以下の3ステップで回答。
    1. ユーザーの真の意図（ゴール）
    2. 技術的課題（解決すべき点）
    3. 修正対象の具体的ファイルパスと修正内容の仮説

## search_impact

修正対象に関連する依存関係を徹底的に調査します。

- **Process**:
    1. 修正対象のクラス/関数名を `grep` や `ripgrep` でプロジェクト全体から検索。
    2. 型定義（`*.d.ts` や `interfaces`）への影響を確認。
    3. フロントエンド/バックエンドを跨ぐ変更（API等）がないか確認。

## verify

実装内容の妥当性を機械的に検証します。

- **Process**:
    1. `pnpm lint` または `npm run lint` の実行。
    2. `pnpm build` による型チェック。
    3. 関連するユニットテストの実行。
