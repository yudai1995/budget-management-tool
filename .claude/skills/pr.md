# /pr

このスキルは、現在の作業ブランチの変更内容を GitHub にプッシュし、プルリクエストを作成します。

## 実行プロセス

1. **プッシュの実行**
    - `git push origin HEAD` を実行する。

2. **PR情報の生成**
    - タイトル: 最新のコミットメッセージ、または `/translate` で定義した技術的課題から生成する。
    - 本文: 以下の項目を網羅して生成する。
        - 【概要】: 今回の修正の目的。
        - 【変更点】: 修正したファイルと変更内容の箇条書き。
        - 【検証結果】: 実行したテストやビルドの結果。

3. **PR作成の実行**
    - `gh pr create --title "{title}" --body "{body}" --draft` を実行する。
    - ※ 初期状態は `--draft`（ドラフト）として作成し、最終確認をユーザーに委ねる。

4. **報告**
    - 作成された PR の URL をユーザーに提示する。

## 制約

- `gh` コマンドがインストールされていない場合は、エラーを報告しインストールを促す。
- すでに PR が存在する場合は、その URL を表示するのみとする。
