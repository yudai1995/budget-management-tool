# /commit

このスキルは、ユーザーからの明示的な指示（「/commit」等）に基づき、差分の妥当性を検証した上で実行される。

## 実行プロセス

1. **差分の網羅的検証**
    - `git status` および `git diff` を実行し、全ての変更箇所を網羅的に確認する。
    - 変更内容を論理的なまとまり（Atomic Commit）として整理する。

2. **ブランチ整合性のチェック（重要）**
    - 現在の作業ブランチ名（例: `feat/xxx`, `fix/yyy`）と実際の差分内容を照らし合わせる。
    - **不適切な差分の検知**:
        - `feat` ブランチなのにバグ修正（`fix`）が混入している。
        - 本来のタスクとは無関係なリファクタリングが含まれている。
    - **アクション**:
        - 不適切な差分が存在する場合、コミットを中断し、別ブランチでの作業や `git stash` を用いた整理をユーザーに推奨する。

3. **メッセージの生成（規約遵守）**
    - `CLAUDE.md` の運用ルールに従い、`{type}: {description}` 形式で作成する。
    - 変更理由と内容を、事実に基づき日本語で網羅的に記述する。

4. **実行と報告**
    - 整合性が確認された差分のみを `git add` し、コミットを実行する。
    - 実行後、ハッシュ値、適用したメッセージ、および整理した差分の概要を報告する。

## 制約

- 整合性チェックをパスしない限り、無理にコミットを進めてはならない。
- `git push` はユーザーからの別途指示を待つこと。
