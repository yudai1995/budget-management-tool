# UIワイヤーフレーム・操作フロー定義

> レガシー React アプリ（`client/`）の解析結果。新フロントエンド（`apps/web/`）への移植基準として使用する。

---

## 1. 画面構成・ルーティング

```
/login              → ログイン画面
/                   → ホーム（月次カレンダー + クイック入力）
/monthly            → 月次カレンダー（/ と同一）
/edit               → 支出登録編集（カレンダーの日付クリックで遷移）
/report             → レポート画面
*                   → 404 Not Found
```

### 認証ガード
- 非認証状態で `/login` 以外にアクセス → `/login` へリダイレクト
- 認証成功後 → `/` へリダイレクト
- ゲストログイン機能あり（ID: `guest-en`、表示名: `guest-ja`）

## 3. ログイン画面（`/login`）

```
┌──────────────────────────────┐
│         ログインフォーム       │
│  ┌──────────────────────┐   │
│  │ ユーザーID             │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ パスワード             │   │
│  └──────────────────────┘   │
│  [ログイン]  [ゲストログイン]  │
│  <エラーメッセージ表示域>      │
└──────────────────────────────┘
```

**インタラクション**
- ID・パスワード両方入力されるまでログインボタン無効化（`disabled`）
- POST `/api/login` → accessToken/userId をストアに保存
- エラー時: ステータスコード別エラーメッセージ表示
- ゲストログイン: 専用 API エンドポイントへ POST

## 4. ホーム・月次カレンダー（`/` , `/monthly`）

```
┌─────────────────┬───────────────────────────────┐
│   SubColumn     │       MainColumn               │
│  (40% 幅)       │       (60% 幅)                 │
│                 │                               │
│ ┌─────────────┐ │ ┌─────────────────────────┐  │
│ │ クイック入力 │ │ │   月次カレンダー          │  │
│ └─────────────┘ │ │   (FullCalendar)         │  │
│ ┌─────────────┐ │ │                          │  │
│ │ 今日のサマリ │ │ │ 各日付セルに支出/収入合計  │  │
│ └─────────────┘ │ │ （色分け表示）            │  │
│ ┌─────────────┐ │ └─────────────────────────┘  │
│ │ 最近の履歴   │ │                               │
│ └─────────────┘ │  [← 前月]          [翌月 →]   │
└─────────────────┴───────────────────────────────┘
```

**カレンダー操作**
- 前月・翌月ボタン: カスタムハンドラで `FullCalendar` API を呼び出し
- 日付クリック: `/edit` へ遷移（`state: { date: Date }` を渡す）
- イベント表示: 支出は赤、収入は青 などで色分け

**SubColumn コンテンツ**
1. **クイック入力フォーム** (`SimpleInputForm`)
   - 収支タイプタブ（支出 / 収入）
   - 金額入力
   - 内容テキスト入力
   - カテゴリ選択（ボタン一覧）
   - 日付ピッカー（`react-datepicker`）
   - 送信ボタン
2. **今日のサマリ** (`TodayReport`): 今日の支出・収入合計
3. **最近の履歴** (`RecentReport`): 直近の支出リスト + グラフ

## 5. 支出登録・編集（`/edit`）

```
┌─────────────────────────────────────────────────┐
│  [支出] [収入]  ← 収支タイプ切り替えタブ          │
│  ┌────────────────────────────────────────┐     │
│  │ 金額                    [数値入力]      │     │
│  │ 内容                    [テキスト入力]  │     │
│  │ カテゴリ  [cat1] [cat2] [cat3] ...     │     │
│  │ 日付                    [日付ピッカー]  │     │
│  └────────────────────────────────────────┘     │
│  [登録する]                                       │
└─────────────────────────────────────────────────┘
```

**バリデーション**
- `class-validator` によるモデルレベルバリデーション
- エラー時: フォームエラー表示、処理中はローディング状態

**遷移元からの状態継承**
- カレンダーの日付クリック → `location.state.date` から日付を初期値設定

## 6. レポート画面（`/report`）

```
┌─────────────────┬───────────────────────────────┐
│   SubColumn     │       MainColumn               │
│                 │                               │
│ ┌─────────────┐ │ ┌──────────────────────────┐ │
│ │レポート設定  │ │ │      グラフ表示            │ │
│ │・期間切り替え│ │ │   (円グラフ / 棒グラフ等) │ │
│ │  [月次|年次]│ │ └──────────────────────────┘ │
│ │・表示種別   │ │ ┌──────────────────────────┐ │
│ │  [全|支出  │ │ │      明細リスト           │ │
│ │   |収入]   │ │ │   日付・カテゴリ・金額     │ │
│ └─────────────┘ │ └──────────────────────────┘ │
└─────────────────┴───────────────────────────────┘
```

**フィルタリング条件**
- レポートタイプ: `月次` / `年次`
- 対象日付: 月次→年月選択、年次→年選択
- 収支種別: `全て (2)` / `支出 (0)` / `収入 (1)`

**合計値表示** (`TotalValueReport`)
- フィルタ条件に一致する合計金額を大きく表示

## 7. 状態管理（Redux Slices）

| Slice | 役割 |
|---|---|
| `budgetListSlice` | 支出・収入レコード一覧。`sumAmount()`・`getTargetDateList()` などのセレクタ |
| `CategoryListSlice` | カテゴリ一覧・選択中カテゴリ |
| `LoginStateSlice` | ログインユーザー情報・エラーメッセージ |
| `FetchingStateSlice` | API 通信中フラグ（ローディング制御） |
| `ReportState` | レポート画面の表示条件（期間タイプ・対象日付・収支種別） |

## 9. UX要件（移植時の必須要件）

- **ローディング状態**: API 通信中は操作を無効化し、インジケーターを表示する
- **エラー状態**: API エラー・バリデーションエラーを画面上で明示する
- **破壊的操作**: 削除は確認ダイアログを必ず挟む
- **レスポンシブ**: モバイル対応（2カラムレイアウトはモバイルで縦積みにする）
- **キーボード操作**: フォーム送信は `Enter` キーでも可
