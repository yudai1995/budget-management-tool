# プロダクト・アイデンティティ

## ミッション

**家計が苦しい家庭が、数字を見て動けるようになる道具を作る。**

記録を習慣化させ、支出を抑える行動変容を促す。
お金を使わせず、アプリを使わせる。

---

## ターゲットユーザー

| 項目 | 内容 |
|------|------|
| ペルソナ | 一般家庭の主婦（20〜50代） |
| 状況 | 家計が月々ギリギリ、または赤字が続いている |
| 目的 | 家計を安定させたい。でも何から始めればいいか分からない |
| リテラシー | 家計簿アプリの経験あり、継続が苦手 |

---

## プロダクトの立ち位置

### やること

- **現在地を数字で見せる** — 今の支出ペースで家計があと何ヶ月もつかを表示する
- **記録に即時フィードバックを返す** — 入力するたびに数字が変わる体験を作る
- **比較で気づかせる** — 統計データとの対比で「自分の支出が多いカテゴリ」を客観的に示す
- **継続できる設計** — 入力が速く、達成感がある。毎日開きたくなる仕組みを作る

### やらないこと

- **消費を促す機能** — 広告、ポイント、クーポン連携など、支出を増やす機能は一切作らない
- **不安を煽る表現** — 「破綻」「危険」など恐怖を使ったコピーは使わない
- **根拠のない励まし** — 数字に基づかない「よくできました」は表示しない
- **多機能化** — 家計の安定に直結しない機能は追加しない

---

## エンゲージメント設計（手放せなくなる仕組み）

ユーザーが継続して使うために、以下の心理メカニズムを設計に組み込む。

| メカニズム | 実装方法 |
|-----------|---------|
| **進捗の可視化** | 「家計の寿命」が日々変化することで、記録行動に即時の意味を与える |
| **連続記録の強化** | 支出ゼロの日が続くと専用メッセージを表示する |
| **相対的な位置づけ** | 偏差値で「自分が平均よりどこにいるか」を示し、改善目標を具体化する |
| **小さな達成感** | 入力ステップを3タップに最小化し、完了体験を毎日積ませる |

これらは「もっとお金を使いたくなる」ではなく「もっと記録して節約したくなる」方向に設計する。

---

## トーン＆ボイス

| 原則 | 具体的な適用 |
|------|------------|
| **事実を伝える** | 数字は正確に、楽観・悲観のバイアスを加えない |
| **温かく、フラットに** | 家計が苦しいユーザーを責めない。状況を伝えるだけ |
| **簡潔に** | 文章は短く。ユーザーに読ませるより、数字を見せる |
| **押しつけない** | 「〜すべき」「〜しなければ」の命令表現は使わない |

---

## デザイン原則

- **メインカラー**: `#f08030`（オレンジ）— 温かみと行動を促す色
- **背景**: `#fef5ee`（ウォームベージュ）— 安心感
- **角丸**: `--radius-card: 12px` を基本
- **アイコン**: `lucide-react` を使用。絵文字は使用しない
- **数値フォント**: モノスペース系で視認性を確保

## AI開発ガイドライン

新機能・UI・文言を提案・実装する前に以下の3点を確認する。**すべて満たさない場合は実装しない。**

| 確認項目 | 問い |
|---------|------|
| **目的との一致** | その機能は「家計を安定させる」ことに直接寄与するか？ |
| **ユーザー負荷** | 主婦が日常の流れで使えるか。操作が増えていないか？ |
| **誠実さ** | 数字や事実を隠したり、誇張したりしていないか？ |

### 判定例

| 提案 | 判定 | 理由 |
|------|------|------|
| 支出ゼロ日の連続カウント表示 | OK | 継続動機につながる |
| ポイント還元率の比較表示 | NG | 支出を促す可能性がある |
| 「よく頑張りました」の自動メッセージ | NG | 数字に基づかない |
| カテゴリ別の節約提案（統計比較） | OK | 行動変容に直結する |
